وثيقة التحليل الفني التفصيلي (SDD)

تحليل دقيق للعمليات الوظيفية (Functional Processes) وقواعد البيانات والواجهات، متضمناً ميزات: سوق المنتجات (Click & Collect) و نظام حجز الأموال (Escrow).

خوارزمية نظام الضامن المالي (Escrow Logic)

هذه الآلية البرمجية تحكم حركة الأموال عند شراء منتج ملموس (لابتوب/جوال). الهدف هو ضمان حق الطرفين.

  • العميل يدفع، المبلغ يذهب إلى (Holding Wallet).
  • التاجر يرى المبلغ كـ (Pending Balance).
  • عند مسح كود الاستلام (Pickup QR)، يتحول المبلغ فوراً إلى (Available Balance).
  • إذا لم يتم الاستلام خلال 48 ساعة (أو حسب الإعداد)، يتاح خيار "نزاع" أو "استرجاع".

أولاً: لوحة تحكم الأدمن (Admin Panel)

إدارة المنتجات (الموافقة والمراقبة)
  • قائمة المنتجات المعلقة Approve/Reject Products
  • تفاصيل المنتج الصور، المخزون، السعر الأصلي، سعر العرض
  • التحكم في العرض تثبيت في الرئيسية، تمييز (Featured)
  • سجل الحركات (Audit Log) تتبع تغييرات المخزون والأسعار
الإدارة المالية والضامن (Escrow)
  • رصيد الضمان الكلي Total Held Funds
  • تسوية المنازعات Dispute Resolution (Refund/Release)
  • سجل السحوبات Withdrawal Requests (Merchants)
  • تقرير العمولات نسبة التطبيق من المبيعات
إدارة قسائم الخدمات
  • إنشاء/تعديل العرض Service Deal CRUD
  • إعدادات الصلاحية Expiry Date, Redemption Rules
  • تقارير الاستخدام Redeemed vs Expired Vouchers

الهيكلية التقنية (Technical Architecture)

قاعدة البيانات (Database Schema Snippets)

// Products Table
Product {
  id: UUID,
  merchant_id: UUID,
  stock_qty: Integer,  // Critical for locking
  is_physical: Boolean // True for Products
}

// Orders Table (Escrow Support)
Order {
  id: UUID,
  user_id: UUID,
  total_amount: Decimal,
  payment_status: 'HELD_IN_ESCROW',
  delivery_status: 'READY_FOR_PICKUP',
  escrow_release_date: Timestamp (Nullable)
}

API Endpoints (Examples)

  • POST /api/v1/orders/create (Atomic Transaction)
  • POST /api/v1/merchant/scan-pickup // Triggers funds release
  • GET /api/v1/user/my-purchases?type=product
  • POST /api/v1/ai/generate-description (Gemini)
